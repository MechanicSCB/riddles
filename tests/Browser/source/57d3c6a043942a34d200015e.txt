<h2 id="lesson">Lesson</h2>
<p>Prior to PHP 5.6.x, if a function were to receive a variable number of arguments, the only way to access them was to use certain special built-in PHP functions such as <code>func_get_args()</code> which returns the argument list of a function in the form of an array.  Such a method of accessing the argument list could be considered unnatural (or at least that's how I feel about it :p) as an amateur developer would think that said function did not receive any arguments just by looking at the function declaration:</p>
<pre><code class="language-php"><span class="cm-comment">// Accessing an argument list of variable length prior to PHP 5.6.x</span>
<span class="cm-keyword">function</span> <span class="cm-def">foo</span>() {
  <span class="cm-comment">// Notice how the function declaration itself (function foo()) looks as if foo() does not accept any arguments</span>
  <span class="cm-variable-2">$argument_list</span> <span class="cm-operator">=</span> <span class="cm-builtin">func_get_args</span>(); <span class="cm-comment">// $argument_list now contains the argument list in the form of an array</span>
  <span class="cm-builtin">var_dump</span>(<span class="cm-variable-2">$argument_list</span>); <span class="cm-comment">// Print out the argument list</span>
}

<span class="cm-variable">foo</span>(<span class="cm-number">1</span>, <span class="cm-number">2</span>, <span class="cm-number">3</span>, <span class="cm-number">4</span>, <span class="cm-number">5</span>); <span class="cm-comment">// The structure of array(1, 2, 3, 4, 5) is printed to the screen</span></code></pre>
<p>Fortunately, PHP 5.6.x introduces the <strong>splat operator</strong>.  The splat operator allows the PHP developer to specify that a function will be receiving a variable number of arguments right in the function declaration instead of in the function body.</p>
<p>The syntax for the spread operator is three dots <code>...</code> followed by the name of the parameter that stores the argument list (as an array).  For example:</p>
<pre><code class="language-php"><span class="cm-comment">// PHP 5.6.x+ Syntactical Sugar</span>
<span class="cm-keyword">function</span> <span class="cm-def">foo</span>(...<span class="cm-variable-2">$arguments</span>) {
  <span class="cm-comment">// The argument list is now automatically stored in $arguments as an array</span>
  <span class="cm-builtin">var_dump</span>(<span class="cm-variable-2">$arguments</span>); <span class="cm-comment">// Prints out the argument list</span>
}

<span class="cm-variable">foo</span>(<span class="cm-number">1</span>, <span class="cm-number">2</span>, <span class="cm-number">3</span>, <span class="cm-number">4</span>, <span class="cm-number">5</span>); <span class="cm-comment">// Prints out the structure of array(1, 2, 3, 4, 5) to the screen</span></code></pre>
<p>Note that the splat operator can actually be used <strong>after a fixed number of arguments</strong> (for example, to distinguish the first argument of a function from the rest) which was previously not possible using built-in functions such as <code>func_get_args()</code>:</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">bar</span>(<span class="cm-variable-2">$first_arg</span>, ...<span class="cm-variable-2">$other_args</span>) {
  <span class="cm-builtin">var_dump</span>(<span class="cm-variable-2">$first_arg</span>); <span class="cm-comment">// Prints out the first argument</span>
  <span class="cm-builtin">var_dump</span>(<span class="cm-variable-2">$other_args</span>); <span class="cm-comment">// Prints out the rest of the arguments in the form of an array</span>
}

<span class="cm-variable">bar</span>(<span class="cm-string">"</span><span class="cm-string">Hello World"</span>, <span class="cm-number">1</span>, <span class="cm-number">4</span>, <span class="cm-number">9</span>, <span class="cm-number">16</span>, <span class="cm-number">25</span>);
<span class="cm-comment">/* The structure of "Hello World" is printed to the screen */</span>
<span class="cm-comment">/* The structure of array(1, 4, 9, 16, 25) is then printed to the screen */</span></code></pre>
<p>Conversely, an array can be passed into a function as an argument list by using the splat operator:</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">add</span>(<span class="cm-variable-2">$a</span>, <span class="cm-variable-2">$b</span>) {
  <span class="cm-keyword">return</span> <span class="cm-variable-2">$a</span> <span class="cm-operator">+</span> <span class="cm-variable-2">$b</span>;
}

<span class="cm-variable">add</span>(...<span class="cm-keyword">array</span>(<span class="cm-number">13</span>, <span class="cm-number">27</span>)); <span class="cm-comment">// $a = 13 and $b = 27 so 40 is returned</span>

<span class="cm-keyword">function</span> <span class="cm-def">multiply_five_numbers</span>(<span class="cm-variable-2">$a</span>, <span class="cm-variable-2">$b</span>, <span class="cm-variable-2">$c</span>, <span class="cm-variable-2">$d</span>, <span class="cm-variable-2">$e</span>) {
  <span class="cm-keyword">return</span> <span class="cm-variable-2">$a</span> <span class="cm-operator">*</span> <span class="cm-variable-2">$b</span> <span class="cm-operator">*</span> <span class="cm-variable-2">$c</span> <span class="cm-operator">*</span> <span class="cm-variable-2">$d</span> <span class="cm-operator">*</span> <span class="cm-variable-2">$e</span>;
}

<span class="cm-variable">multiply_five_numbers</span>(...<span class="cm-keyword">array</span>(<span class="cm-number">1</span>, <span class="cm-number">2</span>, <span class="cm-number">3</span>, <span class="cm-number">4</span>, <span class="cm-number">5</span>)); <span class="cm-comment">// Returns 120 since $a = 1, $b = 2 and so on</span></code></pre>
<p>Note, however, that it is <em>not</em> possible to place the splat operator <strong>before</strong> a fixed number of arguments:</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">baz</span>(...<span class="cm-variable-2">$args</span>, <span class="cm-variable-2">$x</span>, <span class="cm-variable-2">$y</span>, <span class="cm-variable-2">$z</span>) {
  <span class="cm-comment">// Will not work and probably throw an error</span>
}</code></pre>
<h2 id="task">Task</h2>
<p><em>Note: The lesson provided in this Kata is designed to teach you most, if not all, of the key concepts required to complete the Task in this Kata.  However, if in doubt, you are strongly encouraged to conduct your own research.</em></p>
<p>Define the following functions, using the splat operator when necessary (you <em>could</em> technically use <code>func_get_args()</code> instead, but hey, why not use the more readable syntactical sugar provided by PHP 5.6.x+?).</p>
<ol>
<li><code>sum_all</code> - This function should receive a <strong>variable number of arguments</strong>, all of which are expected to be numbers, and return the sum of all the numbers.</li>
<li><code>ultimate_product</code> - This function should also receive a variable number of arguments, all of which are expected to be numbers.  It should return the product of all the numbers.</li>
<li><code>scale_sum</code> - This function should receive a variable number of arguments.  It should first sum up all of the arguments <strong>except the first argument</strong> and then enlarge the entire sum by the value specified by the <strong>first argument</strong>.  For example, <code>scale_sum(5, 2, 3, 4, 5) === 5 * (2 + 3 + 4 + 5) === 70</code></li>
</ol>
