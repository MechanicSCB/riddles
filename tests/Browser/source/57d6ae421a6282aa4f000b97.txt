<h2 id="about-this-kata-series">About this Kata Series</h2>
<p>"PHP Functions" is a Kata Series authored by <strong>donaldsebleung</strong> which focuses on the perks and interesting features of PHP functions.  It is <em>un-numbered</em> which means that the Kata in this Series is <strong>not</strong> progressive - one kata in this Series does not necessarily depend on another.  This means that as long as you know how to write "Hello World" level programs in PHP, you will probably find any Kata in this Series easy to pick up and complete.</p>
<p>There is, however, one main prerequisite.  You must be sufficiently familiar with primitive data types in PHP (strings, booleans, floats, integers) and what they mean.  You must also have a <strong>basic</strong> understanding of how to define a simple function in PHP that may or may not receive a <strong>fixed</strong> number of arguments.  A good indicator that you are ready for any Kata in this Series is if you are able to complete <a href="https://www.codewars.com/kata/multiply/train/php" target="_blank">Multiply (8kyu) in PHP</a> without looking at external resources in the process.  If you are unable to complete that Kata without researching PHP function syntax then you may have to practice with basic PHP functions until you are familiar with them before continuing on this Kata series.</p>
<p>Certain Kata in this Series may have additional prerequisites.  If that is the case, the extra prerequisites will be listed under a Level 2 Heading called <strong>Prerequisites</strong> in said Kata.  Additionally, if a certain Kata in this Series requires a thorough understanding of PHP, the kata may be labelled as <strong>[Advanced]</strong>.</p>
<h2 id="prerequisites">Prerequisites</h2>
<p>Certain parts of the Task in this Kata require you to be aware of the existence of <strong>classes and objects</strong> in PHP, what they are, what class/object <strong>properties</strong> are and the syntax for accessing properties of an object.</p>
<p>You, however, do <strong>not</strong> need to know they syntax of defining a class and/or its respective properties in this Kata.</p>
<h2 id="lesson">Lesson</h2>
<p>PHP, by default, is a dynamically typed programming language which means that in the vast majority of cases, a function can receive arguments of any type and can return a value of any type.  For example:</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">custom_print</span>(<span class="cm-variable-2">$input</span>) {
  <span class="cm-comment">// Theoretically, $input can be of any type, be it a boolean, integer, float, string, array or object</span>
  <span class="cm-keyword">echo</span> <span class="cm-string">"</span><span class="cm-string">The input is: "</span>;
  <span class="cm-builtin">var_dump</span>(<span class="cm-variable-2">$input</span>);
}

<span class="cm-variable">custom_print</span>(<span class="cm-number">1</span>); <span class="cm-comment">// The input is: int(1)</span>
<span class="cm-variable">custom_print</span>(<span class="cm-variable">M_PI</span>); <span class="cm-comment">// The input is: float(3.141592653589793)</span>
<span class="cm-variable">custom_print</span>(<span class="cm-atom">true</span>); <span class="cm-comment">// The input is: bool(true)</span>
<span class="cm-comment">// ... </span>

<span class="cm-comment">/* You get the point.  The $input and return types were not specified so the function custom_print() can receive any type of input as argument and return any type of value it wishes (NULL in this case). */</span></code></pre>
<p>However, as of PHP 7, it is entirely possible to write <strong>strictly-typed PHP</strong> code by specifying the expected argument types in the function definition, much like C and Java.  It is also possible to specify the return type of a function but that is outside the scope of this lesson.</p>
<p>The syntax for strict-typing in PHP is as follows:</p>
<pre><code class="language-php"><span class="cm-comment">// Declare Strict Mode (you do **not** need to know how to do this in this Kata)</span>
<span class="cm-keyword">declare</span>(<span class="cm-variable">strict_types</span><span class="cm-operator">=</span><span class="cm-number">1</span>);

<span class="cm-comment">// Function Definition - notice the type declaration for the $name parameter</span>
<span class="cm-keyword">function</span> <span class="cm-def">greet</span>(<span class="cm-variable">string</span> <span class="cm-variable-2">$name</span>) {
  <span class="cm-keyword">return</span> <span class="cm-string">"</span><span class="cm-string">Hello </span><span class="cm-variable-2">$name</span><span class="cm-string">"</span>;
}

<span class="cm-comment">// Executing the function with different arguments</span>
<span class="cm-variable">greet</span>(<span class="cm-string">"</span><span class="cm-string">Donald"</span>); <span class="cm-comment">// "Hello Donald"</span>
<span class="cm-variable">greet</span>(<span class="cm-string">"</span><span class="cm-string">Ella"</span>); <span class="cm-comment">// "Hello Ella"</span>
<span class="cm-variable">greet</span>(<span class="cm-string">"</span><span class="cm-string">Lucy"</span>); <span class="cm-comment">// "Hello Lucy"</span>
<span class="cm-variable">greet</span>(<span class="cm-number">3.14</span>); <span class="cm-comment">// TypeError thrown - 3.14 is not a string</span></code></pre>
<p>As seen from the example above, the exact syntax for declaring the expected type of a function argument is the name of the expected type, followed by the parameter name.  For example, if your function receives exactly one argument <code>$n</code> which is expected to be an <strong>integer</strong>, you would type the following: <code>int $n</code>.</p>
<p>The following keywords are used for each data type:</p>
<ol>
<li><code>int</code> for Integer</li>
<li><code>float</code> for Float (N.B. If an integer is passed in instead, PHP will coerce it into a float instead of throwing a TypeError)</li>
<li><code>bool</code> for Boolean</li>
<li><code>string</code> for String</li>
<li><code>array</code> for Array (available from PHP 5.1.0+) (<strong>NOTE: You are NOT expected to know this and will not be tested on it in this Kata</strong>)</li>
<li><code>callable</code> for Callable, e.g. a closure or the name of a function (available from PHP 5.4.0+) (<strong>NOTE: You are NOT expected to know this and will not be tested on it in this Kata</strong>)</li>
<li>The name of a class is used when expecting an instance of that class, e.g. <code>Animal $animal</code> if the argument <code>$animal</code> is expected to be an instance of <code>Animal</code> (<em>we will come back to that shortly - read on</em>)</li>
</ol>
<p>Note that, like C and Java, (argument) type declarations are done on an <strong>argument by argument basis</strong>.  This means that the following is valid PHP:</p>
<pre><code class="language-php"><span class="cm-comment">// Strict Mode</span>
<span class="cm-keyword">declare</span>(<span class="cm-variable">strict_types</span><span class="cm-operator">=</span><span class="cm-number">1</span>);

<span class="cm-comment">// Function Definition</span>
<span class="cm-keyword">function</span> <span class="cm-def">some_function</span>(<span class="cm-variable">string</span> <span class="cm-variable-2">$s</span>, <span class="cm-variable">float</span> <span class="cm-variable-2">$f</span>) {
  <span class="cm-keyword">return</span> <span class="cm-string">"</span><span class="cm-string">The string passed in is </span><span class="cm-variable-2">$s</span> <span class="cm-string">and the float passed in is </span><span class="cm-variable-2">$f</span><span class="cm-string">"</span>;
}

<span class="cm-comment">// Trying out some function calls</span>
<span class="cm-variable">some_function</span>(<span class="cm-string">"</span><span class="cm-string">Dog"</span>, <span class="cm-number">3.14</span>); <span class="cm-comment">// "The string passed in is Dog and the float passed in is 3.14"</span>
<span class="cm-variable">some_function</span>(<span class="cm-string">"</span><span class="cm-string">Cat"</span>, <span class="cm-number">2.5</span>); <span class="cm-comment">// "The string passed in is Cat and the float passed in is 2.5"</span>
<span class="cm-variable">some_function</span>(<span class="cm-number">3.14</span>, <span class="cm-string">"</span><span class="cm-string">Cat"</span>); <span class="cm-comment">// TypeError is thrown</span>
<span class="cm-variable">some_function</span>(<span class="cm-string">"</span><span class="cm-string">Donald"</span>, <span class="cm-string">"</span><span class="cm-string">Computer Programmer"</span>); <span class="cm-comment">// TypeError is thrown</span></code></pre>
<p>Finally, note that apart from the primitive data types <code>bool</code>, <code>int</code>, <code>float</code> and <code>string</code>, your function can also expect <strong>an instance of a particular class</strong> as an argument, e.g.</p>
<pre><code class="language-php"><span class="cm-comment">// Strict Mode</span>
<span class="cm-keyword">declare</span>(<span class="cm-variable">strict_types</span><span class="cm-operator">=</span><span class="cm-number">1</span>);

<span class="cm-comment">// Class Declarations and Definitions (you will NOT be tested on these in this Kata)</span>
<span class="cm-keyword">class</span> <span class="cm-def">Animal</span> {
  <span class="cm-comment">/* Class code here */</span>
}
<span class="cm-keyword">class</span> <span class="cm-def">Plant</span> {
  <span class="cm-comment">/* Class code here */</span>
}
<span class="cm-keyword">class</span> <span class="cm-def">Dog</span> <span class="cm-keyword">extends</span> <span class="cm-variable">Animal</span> {
  <span class="cm-comment">/* Class code here.  An instance of `Dog` is also an instance of `Animal`. */</span>
}

<span class="cm-comment">// Our Function</span>
<span class="cm-keyword">function</span> <span class="cm-def">some_function_expecting_an_animal</span>(<span class="cm-variable">Animal</span> <span class="cm-variable-2">$a</span>) {
  <span class="cm-comment">// $a is expected to be an instance of the class `Animal`</span>
  <span class="cm-comment">/* Function code here */</span>
}

<span class="cm-comment">// Invoking the function with different arguments</span>
