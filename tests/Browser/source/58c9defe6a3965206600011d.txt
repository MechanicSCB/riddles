<h2 id="about-this-kata-series">About this Kata Series</h2>
<p>This Kata Series explores a core API in PHP called <a href="http://php.net/manual/en/book.reflection.php" target="_blank">Reflection</a> which allows a developer to <em>reverse-engineer</em> classes, interfaces, functions, methods and extensions.  It is assumed that the user undertaking this Kata Series is already familiar with both functional and object-oriented programming in PHP.  A good indicator that you are ready for this Kata Series is if you are able to complete at least <strong>6 out of 8</strong> topics/Kata in my <a href="https://www.codewars.com/collections/php-functions" target="_blank">PHP Functions</a> series without hesitation and are at least moderately familiar with all the topics covered in the <strong>first 7 Kata</strong> in my <a href="https://www.codewars.com/collections/object-oriented-php" target="_blank">Object-Oriented PHP</a> series.  Certain Kata in this Series may also require slightly more advanced OOP knowledge such as the awareness of what an interface is, but if such extra knowledge is required, it will be mentioned under a "Prerequisites" subtitle.</p>
<p>Most Kata in this Series will consist of two main parts - a "Lesson" and a "Task".  However, unlike either of "Object-Oriented PHP" and "PHP Functions", the "Lesson" will only teach the fundamental concepts required to <strong>understand</strong> the "Task" presented in each Kata.  In most cases, you will be expected to look up on <a href="http://php.net/manual/en/book.reflection.php" target="_blank">the official Reflection documentation</a> in order to figure out the details of how to actually complete the Kata "Task" (e.g. which methods to call or even what class to use!).  If you are still in doubt after referring to the official documentation, you are <strong>strongly encouraged</strong> to conduct your own research and use whatever resources are available to you on the Internet or otherwise.</p>
<h2 id="lesson">Lesson</h2>
<p>In the previous Lesson, we learned that we could extract basic information about the signature (i.e. declaration) of a function through the use of <code>ReflectionFunction</code> which is part of the Reflection API.  For example, we could obtain information about whether said function is a closure and whether it has declared a return type.  However, one main thing we have not touched upon is the function parameters themselves.  What is the name of each parameter?  Have they declared a type?  Do they default to a given value if not provided?  Enter <code>ReflectionParameter</code>!  According to the official Documentation, this class also has a constructor which accepts 2 arguments (in order): the name of the function (or the closure itself) and the name of the parameter (without the <code>$</code> symbol) as a string, so it <em>should</em> be technically <em>possible</em> to directly instantiate an instance of <code>ReflectionParameter</code> in our PHP script which will then allow us to study a particular parameter of a function in great detail.  However, the more common (and better documented) method of studying a function's parameters is to first create a <code>ReflectionFunction</code> of said function and then calling on its <code>getParameters()</code> method to obtain an array of <code>ReflectionParameter</code>s where each instance of <code>ReflectionParameter</code> corresponds to a parameter in the function signature.  For example (<em>Note: The</em> <code>&lt;pre&gt;&lt;/pre&gt;</code> <em>tags are just for styling for them to display properly in my browser</em>):</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">multiply</span>(<span class="cm-variable-2">$a</span>, <span class="cm-variable-2">$b</span>) {

<p>Before we move on to our Task in this Kata, you may find the official documentation on <a href="http://php.net/manual/en/class.reflectionparameter.php" target="_blank">ReflectionParameter</a> useful as it contains an introduction to the class itself and contains links to each and every one of its properties/methods/constants.</p>
<h2 id="task">Task</h2>
<p>Write a function <code>filter_parameters()</code> which accepts exactly 1 argument <code>$fn</code> (the name of the function or the closure itself) and returns a string representation of a list containing the names of the parameters of said function that satisfies <strong>all</strong> of the conditions mentioned below:</p>
<ul>
<li>It has declared a data type</li>
<li>Its declared data type is an <strong>integer</strong></li>
<li>It has a <strong>default value</strong></li>
<li>If not provided, it defaults to a value of <code>5</code></li>
</ul>
<p>The returned string should be of the format <code>"item1, item2, item3, ..."</code>, i.e. each parameter name in the list should be separated by the delimiter <code>", "</code></p>
<p>For example, for the following function:</p>
<pre><code class="language-php"><span class="cm-keyword">function</span> <span class="cm-def">example</span>(<span class="cm-variable-2">$a</span>, <span class="cm-variable">int</span> <span class="cm-variable-2">$b</span>, <span class="cm-variable">bool</span> <span class="cm-variable-2">$c</span>, <span class="cm-variable">int</span> <span class="cm-variable-2">$d</span> <span class="cm-operator">=</span> <span class="cm-number">5</span>, <span class="cm-variable">float</span> <span class="cm-variable-2">$e</span> <span class="cm-operator">=</span> <span class="cm-number">5.00</span>, <span class="cm-variable">int</span> <span class="cm-variable-2">$f</span> <span class="cm-operator">=</span> <span class="cm-number">5</span>, <span class="cm-variable">int</span> <span class="cm-variable-2">$g</span> <span class="cm-operator">=</span> <span class="cm-number">15</span>) { <span class="cm-comment">/* ... */</span> }</code></pre>
<p>... your function should return <code>"d, f"</code>.  This test case has been included for you.  You may assume that any function (or name thereof) passed into your function will have at least one parameter that satisfies all the conditions given.</p>
